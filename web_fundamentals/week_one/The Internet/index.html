Clients and Servers

Computer connected to the Web are called clients and servers

  • Clients are the typical Web user's internet connected devices(for example, your computer connected to your WI-IF, or your phone connected to your mobile network) and Web-accessing software available on those devices(usually a web browser like Firefox or Chrome).

  • Servers are computer's that store webpages, sites, or apps. When a client device wants to access a webpage, a copy of the webpage is downloaded from the server onto the client machine to be displayed in the user's web browser.

So What Actually Happens?

Computer connected to the Web are called clients and servers

  1. The browser goes to the DNS server and finds the address of the server that the website lives on.

  2. The browser sends an HTTP request message to the server asking it to send a copy of the website to the client

  3. If the server approves the client's request, the server sends the client a "200 OK" message, which means "Permission granted!Here you go!", and then starts sending the website files to the browser as a series of small chunks.

  4. The browser assembles the small chunks into a complete website and displays it to you!

The Big 3- HTML, CSS, JavaScript

While the server can process information in many different languages, the files that they serve to the clients are always going to be some combination of HTML, CSS, and JavaScript!
